# Epictetus Operating Model — David Allen, *Getting Things Done: The Art of Stress-Free Productivity*

> Source frame: your **mind is for having ideas, not holding them**. Modern knowledge work has no edges, morphs constantly, and is always-on, producing **ambient angst** — the physiological cost of carrying **open loops** (unmanaged commitments) in mental **RAM**. The cure is not working harder but **appropriate engagement**: get 100% of "stuff" out of your head into a **trusted system**, then run the five-stage workflow — **Capture → Clarify → Organize → Reflect → Engage**. Clarity comes from defining outcomes ("what does *done* look like?") and the very next **physical, visible action** ("what does *doing* look like?"). The reward is **mind like water**: a state of relaxed control that responds to input appropriately — no overreaction, no underreaction — so you can be fully present.

## Layer 1 — Practice (What must be done?)

### What we’re trying to achieve
- Empty the head: zero open loops carried in RAM.
- Reach **mind like water** — relaxed control, fully present with the task in front of you.
- Convert "amorphous blobs of undoability" into discrete next actions and defined outcomes.
- Trust the system 100% so the mind stops re-reminding (no leaky bucket).

### Non-negotiable behaviors
- **Capture everything**: 100% of incompletes go into a container outside your head, never back into memory.
- **Define the next action**: every actionable item gets a *physical, visible* next step before it leaves the in-tray.
- **Honor the Two-Minute Rule**: if it takes under two minutes, do it now during clarifying.
- **Keep the calendar sacred**: only the **Hard Landscape** (time-specific, day-specific actions/information) lives there.
- **Run the Weekly Review**: the master key — non-negotiable, or the system decays and angst returns.
- **Clarify outcome + next action for every project**: define "done" and "doing."

### Routines / checklists

- **Capture sweep (whenever attention is snagged)**
  - Dump the thought/email/commitment into a capture tool (in-tray, pad, app).
  - Minimize the number of capture buckets; empty them regularly.
  - Never evaluate while capturing — just collect.

- **Clarify pass (process the in-tray to empty)**
  - Ask: *What is it?* Then: *Is it actionable?*
  - If **No** → Trash, Incubate (Someday/Maybe or Tickler), or Reference (file it).
  - If **Yes** → name the **Next Action** (next physical, visible behavior).
    - Under 2 minutes? Do it now.
    - Delegate it? Track on **Waiting For**.
    - Defer it? Put on a context Next Actions list or the calendar.
  - Multi-step outcome? Add it to the **Projects List**.

- **Organize the buckets**
  - **Projects List**: every outcome needing >1 step, finishable within a year.
  - **Calendar**: Hard Landscape only.
  - **Next Actions lists by context**: @Calls, @Computer, @Office, @Errands, @Home.
  - **Waiting For**: delegated outcomes with dates.

- **Natural Planning Model (vertical/project thinking)**
  1. **Purpose & Principles** — define the "why" and the standards.
  2. **Outcome Visioning** — what does success look, sound, feel like?
  3. **Brainstorming** — distributed cognition; get ideas onto paper.
  4. **Organizing** — components, sequences, priorities.
  5. **Identify Next Actions** — allocate the next physical move.

- **Engage (choosing in the moment) — Four-Criteria Model**
  - **Context**: what can I do *here*, with these tools?
  - **Time Available**: how long until the next commitment?
  - **Energy Available**: physical/mental horsepower right now?
  - **Priority**: highest payoff among the remaining options.

- **Weekly Review (the master key, ~60 min)**
  - **Get Clear**: collect loose papers, empty head, process all in-trays to zero.
  - **Get Current**: review Next Actions, Calendar (past + upcoming), Waiting For, Projects List.
  - **Get Creative**: review Someday/Maybe; add new ideas.
  - Goal state: *clean, clear, current, and complete*.

### What good execution looks like
- The head is empty; nothing nags because everything is captured.
- Every project on the list has a defined outcome and a clear next action.
- The calendar contains only what genuinely must happen on that day/time.
- In-trays hit zero on a regular cadence; the system is trusted 100%.
- You can shift smoothly between tasks (horizontal control) and dive deep when needed (vertical control).

## Layer 2 — Principles (Why does it work?)

- **The mind is a focusing tool, not a storage device**: RAM held by open loops degrades thinking and creates 24/7 pseudo-pressure.
- **Stress comes from unmanaged commitments, not workload**: "anxiety is caused by a lack of control, organization, preparation, and action."
- **You can’t *do* a project — only a next action**: physical, visible behaviors are the only things that move reality.
- **Power is proportional to relaxation**: a tense mind is inefficient; mind like water generates clean responses.
- **A trusted system frees cognition**: externalizing commitments (distributed cognition) buys back high-level thinking.
- **Bottom-up beats top-down**: clear the mundane ground floor first to earn the clarity for higher horizons.
- **What's on the calendar is sacred**: mixing have-tos with should-dos dilutes its authority and forces the brain to take back remembering.

### Decision rules (doctrine)
- If something has your attention, capture it — do not try to "remember" it.
- If it’s actionable and takes under 2 minutes, do it now.
- If it isn’t actionable, decide: trash, incubate, or reference — never leave it as "stuff."
- If a project feels stuck, you haven’t defined the next physical action yet.
- If a task isn’t time- or day-specific, it does not belong on the calendar.
- If the system feels untrustworthy, run a Weekly Review before doing more work.

## Layer 3 — Meta (How do we evaluate and improve?)

### Metrics (leading, behavior-tied)
- **Inbox-to-zero frequency** (how often all capture buckets are emptied).
- **Open-loop capture rate** (% of commitments living in the system vs. the head).
- **Projects with a defined next action** (target: 100%).
- **Weekly Review completion streak** (consecutive weeks done).
- **Two-minute items cleared on contact** (vs. deferred unnecessarily).
- **Calendar purity** (% of calendar entries that are true Hard Landscape).

### Review loop
- **Daily**: scan calendar (Hard Landscape) + relevant context Next Actions lists; capture continuously.
- **Weekly**: the full Weekly Review — Get Clear, Get Current, Get Creative until clean, clear, current, complete.
- **Monthly/Quarterly**: review the **Six Horizons of Focus** — Ground (actions), Projects, Areas of Focus, Goals, Vision, Purpose/Principles — to align daily action with values.

### Core rule guardrail
- If you feel "ambient angst," the system has a leak: something is being stored in your head, not captured. Stop, do a capture sweep, and process to zero. Never let elaborate tools or higher-horizon planning (Layer 3) substitute for the ground-floor discipline of capturing and defining next actions (Layer 1). A leaky bucket destroys trust, and a distrusted system is no system at all.

---

## Operating Sequence

1. **Capture** 100% of stuff into trusted containers.
2. **Clarify** each item: actionable? next action? (2-minute rule applies).
3. **Organize** into Projects, Calendar, Next Actions by context, Waiting For.
4. **Reflect** via the Weekly Review until clean, clear, current, complete.
5. **Engage** using Context, Time, Energy, Priority.
6. Repeat until mind like water is the default state.
